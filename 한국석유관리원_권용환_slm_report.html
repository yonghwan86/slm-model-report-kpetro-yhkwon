01모델 선택 이유

EXAONE-3.5-2.4B-Instruct는 LG AI Research가 2024년 12월 공개한 한국어·영어 이중 언어 instruction-tuned 모델로, 공공 행정 영역에 적용하기에 가장 균형이 잘 잡힌 SLM이다. 선택 근거는 네 가지다. 첫째, 한국어 성능이다. 동일 체급(2~3B대) 글로벌 모델인 Llama 3.2 3B, Qwen 2.5 3B, Gemma 2 2B 대비 한국어 멀티턴 벤치마크 KoMT-Bench에서 7.24점(Llama 3.2 3B는 3.16점), 한국어 추론 벤치마크 LogicKor에서 8.51점(Llama 3.2 3B는 2.86점)으로 압도적 격차를 보인다. 행정 문서·민원 응대처럼 한국어 정확도가 결정적인 도메인에 직접 부합한다. 둘째, 32K 토큰 컨텍스트 길이다. 같은 체급 다수 모델이 4K~16K에 머무는 반면, 32K 컨텍스트는 수십 페이지 분량의 보고서·법령·회의록을 한 번에 입력해 RAG·요약·질의응답 파이프라인을 구성하기에 충분하다. 셋째, 모델 카드와 기술 보고서의 투명성이다. arXiv 2412.04862 기술 보고서가 사전학습 토큰 수(6.5T), GQA 구조, BBPE 토크나이저, decontamination 절차, SFT/DPO 파이프라인까지 명시하고 있어 도입 의사결정에 필요한 근거가 충분하다. 넷째, 경량 배포 친화성이다. 2.4B 파라미터는 GGUF Q4_K_M 양자화 시 약 1.5~2GB 수준으로, 단일 GPU(8GB) 또는 고사양 CPU 환경에서도 추론 가능해 폐쇄망 온프레미스가 필수인 공공 환경에 적합하다. 단, 라이선스가 비상업(NC)인 점은 본 보고서의 한계 항목에서 별도로 다룬다.

03학습 데이터와 튜닝 방식

① 사전학습(Pre-training) — 기술 보고서(arXiv 2412.04862)에 따르면 2.4B 모델은 총 6.5조(6.5T) 토큰으로 사전학습되었으며, 어휘 비율은 한국어 50% · 영어 50%로 균형 있게 구성된다. 학습은 2단계로 진행됐다. 1단계는 다양한 출처의 대규모 일반 도메인 코퍼스를 사용했고, 2단계에서 long-context fine-tuning을 적용해 max_position_embeddings를 8,192에서 32,768로 확장했다(replay 기반 기법으로 catastrophic forgetting을 완화). 다만 구체적인 데이터셋의 출처·규모는 영업비밀로 간주되어 공개되지 않았으며, 기술 보고서는 "AI Compliance review를 거쳐 저작권·지식재산·개인정보 위험을 검토했다"는 수준으로만 기술한다. ② 데이터 정제(Decontamination) — 학습 데이터에 평가 벤치마크의 테스트 셋이 우연히 섞여 들어가는 문제를 막기 위해 substring-level matching(슬라이딩 윈도우 50자, 무작위 샘플 N=10)을 적용했다고 명시한다. ③ Instruction Tuning(SFT) — 800만 건의 웹 코퍼스에서 분류체계(taxonomy) 기반으로 핵심 지식을 추출한 뒤, instruction evolution 기법(WizardLM 계열 방법)으로 난이도를 다양화한 instruction-response 쌍을 합성해 Supervised Fine-Tuning을 수행했다. ④ Preference Optimization — DAA(Direct Alignment Algorithms) 계열인 DPO와 SimPO를 다단계로 적용했다. 다중 모델에서 N개 응답을 샘플링한 뒤 reward model 점수로 chosen/rejected 쌍을 만들고, 두 reward model 간 동의도(agreement)가 임계값 이하인 쌍은 필터링했다. M0(SFT 모델)→M1→M2 순으로 순차 학습하는 staged 파이프라인을 채택해 over-optimization을 완화했다. ⑤ 적용되지 않은(또는 미공개) 기법 — 기술 보고서와 모델 카드 모두 LoRA·QLoRA·RLHF(PPO) 적용 여부는 언급이 없다. 공개 범위 내에서는 풀 파라미터 SFT + DPO/SimPO 조합으로 보는 것이 합리적이다.

05모델 파일 구성

EXAONE-3.5-2.4B-Instruct 저장소(약 9.63 GB, FP32 기준)에 포함된 핵심 파일과 역할은 다음과 같다. config.json(1.04 kB) — 모델 아키텍처의 모든 하이퍼파라미터를 담은 설정 파일. hidden_size(2560), num_layers(30), num_attention_heads(32), num_key_value_heads(8), head_dim(80), intermediate_size(7168), max_position_embeddings(32768), vocab_size(102400), rope_theta(1000000), rope_scaling(llama3 type, factor 8.0), tie_word_embeddings(true) 등을 정의한다. architectures 필드는 "ExaoneForCausalLM"으로 지정돼 있어 추론 시 trust_remote_code=True 설정이 필요하다. configuration_exaone.py / modeling_exaone.py — Transformers 라이브러리에 내장돼 있지 않은 EXAONE 전용 클래스를 정의한 커스텀 코드 파일. AutoConfig·AutoModelForCausalLM 매핑은 config.json의 auto_map 필드에 명시돼 있다. model.safetensors — 실제 가중치 파일. safetensors 포맷은 pickle 기반 .bin 대비 안전하고 로딩이 빠르다. tokenizer.json / tokenizer_config.json — BBPE 토크나이저의 어휘·머지 규칙·특수 토큰·Jinja2 chat template을 담는다. EXAONE 3.5는 "You are EXAONE model from LG AI Research, a helpful assistant." 시스템 프롬프트 사용을 권장하는 chat template이 내장돼 있다. special_tokens_map.json — <|endoftext|>, <|startofturn|> 등 특수 토큰의 의미적 역할 매핑. generation_config.json(134 B) — model.generate() 호출 시의 기본값(eos_token_id, max_new_tokens 기본값 등)을 정의한다. LICENSE(13.8 kB) — EXAONE AI Model License Agreement 1.1 - NC 전문. assets/ — 모델 카드용 이미지 자원. 추가로 같은 컬렉션 내에 AWQ 4비트 양자화 버전과 GGUF 버전(Q8_0/Q6_K/Q5_K_M/Q4_K_M/IQ4_XS)이 별도 저장소로 제공돼 vLLM·llama.cpp·Ollama 환경에 곧바로 투입할 수 있다.

06활용 가능 업무

EXAONE-3.5-2.4B-Instruct는 ① 한국어 정확도 ② 32K 장문 컨텍스트 ③ 경량 배포라는 세 가지 강점이 결합돼, 공공기관·행정 영역에서 다음 시나리오에 직접 적용할 수 있다. (1) 민원 응대 챗봇 / FAQ 자동화 — 자주 묻는 질의에 대한 1차 응답을 자동 생성하고, 사람 상담사에게는 본 모델이 제안한 답변 초안과 근거 문서를 함께 전달하는 human-in-the-loop 구조에 적합하다. KoMT-Bench 7.24점·LogicKor 8.51점은 동급 모델 중 최상위로, 한국어 자연스러움·존댓말·정중도 면에서 그대로 사용 가능한 수준이다. (2) 한국어 RAG(Retrieval-Augmented Generation) 시스템 — 본 모델이 가장 빛을 발하는 영역이다. Ko-LongRAG 벤치마크에서 74.7점, Ko-WebRAG 73.0점으로 동급 외산 모델(Qwen 2.5 3B 40.5/34.7, Llama 3.2 3B 39.3/50.0)을 큰 격차로 앞선다. 32K 컨텍스트에 검색된 법령·고시·내부 규정 청크를 다수 주입하고 출처와 함께 답변을 생성하는 구조로, 정책 자료 Q&A·법령 검색 도우미 구축에 매우 강하다. (3) 내부 문서 요약 — 회의록·감사보고서·정책연구보고서를 32K 토큰 단위로 한 번에 입력해 핵심 결정사항·액션 아이템을 추출. (4) 보고서·공문 초안 작성 보조 — 사용자가 개요만 제시하면 보고서 형식에 맞춘 초안을 생성. SFT 단계의 instruction evolution 학습 효과로 길이·형식 지시 이행 능력이 우수하다(IFEval 73.6점, 동급 최상위권). (5) 폐쇄망 온프레미스 배포 — Q4_K_M GGUF 양자화 시 약 1.5~2GB로, RTX 3060(12GB) 또는 i9 CPU + 16GB RAM 환경에서도 실시간 추론이 가능해 외부망과 분리된 행정 인프라에 직접 올릴 수 있다. (6) 도메인 추가 학습의 베이스 — 행정용어·법률용어·기관 내부 문서로 LoRA 추가 학습을 진행하면 도메인 특화 어시스턴트로 빠르게 발전 가능하다. 단, 위 모든 시나리오는 NC 라이선스 범위 내 연구·POC 단계에 한정되며, 실서비스 도입은 별도 라이선싱 협의가 전제된다.

07한계와 주의사항

(1) 라이선스 제약(가장 중요한 한계) — EXAONE AI Model License Agreement 1.1 - NC는 명시적으로 비상업·연구용으로 사용을 제한한다. 라이선스 3.1조는 매출 발생 가능한 모든 상업적 활용(직·간접 포함)과 본 모델 출력으로 다른 모델을 학습·개선하는 행위를 금지하며, 라이선스 4.2조는 모델 출력물의 권리도 LG가 보유한다고 규정한다. 따라서 행정 서비스에 정식 배포하려면 contact_us@lgresearch.ai를 통한 별도 commercial agreement가 반드시 선행되어야 한다. (2) 안전성·편향 — 동급 대비는 우수, 절대 수치는 보강 필요 — 한국정보화진흥원(NIA)이 제공한 한국어 LLM 신뢰성 벤치마크 평가에서 2.4B 모델은 편향(Bias) 67.4%, 혐오(Hate) 72.2%, 위법(Illegal) 80.3%, 민감(Sensitiveness) 74.0%로 전체 평균 72.2%를 기록했다. 같은 EXAONE 3.5 시리즈에서도 7.8B(85.6%)·32B(87.1%) 대비 명확히 낮은 수치이므로, 민원 등 외부 노출 서비스에 투입하기 전 추가 안전 필터링·red-teaming이 필수다. (3) 환각·사실 오류 — 모델 카드가 직접 명시하듯, 통계 기반 토큰 생성 특성상 의미·문법 오류, 편향, 사실 모순이 발생할 수 있고 학습 시점 이후 정보는 반영하지 못한다. 행정 정보처럼 정확성이 결정적인 영역에서는 RAG와 결합해 1차 응답 근거를 외부 지식베이스로 강제해야 한다. (4) 도메인 특화 미흡 — 일반 도메인 코퍼스로 학습된 범용 모델이므로, 한국 법령·행정용어·민원 분류체계에 대한 즉시성 정확도는 별도 도메인 fine-tuning을 거치지 않으면 한계가 있다. (5) 운영상 주의 — Transformers 표준 아키텍처가 아니어서 추론 시 trust_remote_code=True가 강제되며, 이는 폐쇄망 보안 정책상 별도의 코드 검증·서명 절차를 요구한다. 또한 기본 가중치는 FP32(약 9.6GB)이므로, 실배포 시 BF16/AWQ/GGUF 양자화 버전 사용을 전제로 인프라를 설계해야 한다. (6) 학습 데이터 출처 비공개 — 사전학습 코퍼스 전체 목록·비율·라이선스가 공개되지 않아, 공공조달 단계에서 요구되는 데이터 투명성·저작권 검증 요건을 100% 충족하기 어려울 수 있다.

08종합 의견

EXAONE-3.5-2.4B-Instruct는 "한국어 SLM이 필요한 공공·연구 환경에서 가장 먼저 검토해야 할 후보"라는 것이 본 보고서의 결론이다. 동급 외산 모델 대비 한국어 성능 격차가 명확하고, 32K 컨텍스트와 GGUF·AWQ 사전양자화 지원 덕분에 폐쇄망 온프레미스 환경에서도 실측 가능한 단계까지 빠르게 도달할 수 있다. ① 도입 적합 조건 — (가) 학내·기관 내부 연구·POC 단계, (나) RAG 기반 한국어 정책·법령 Q&A 시스템, (다) 회의록·보고서 요약 자동화, (라) 외부 노출이 제한된 직원용 어시스턴트가 가장 자연스러운 진입점이다. ② 도입 비적합 조건 — (가) 별도 라이선싱 협의 없이 매출이 발생하는 대국민 서비스, (나) 의료·법률 자문 등 환각 시 피해가 큰 고위험 도메인, (다) 데이터 학습 코퍼스 전체 출처 공개가 조달 요건인 사업은 본 모델만으로는 부적합하다. ③ 권장 검증·POC 절차 — 1) 도메인 특화 평가셋 구축 : 자체 민원/법령 100~300건으로 골든셋을 만들고 정확도·관련성·환각률을 측정. 2) RAG 결합 베이스라인 확보 : Ko-LongRAG·Ko-WebRAG 결과가 시사하듯, 본 모델은 RAG 친화적이므로 단독 추론보다 BM25/임베딩 검색과 결합한 답변 품질을 측정해야 한다. 3) 안전 필터 레이어 추가 : NIA 신뢰성 벤치마크 결과를 근거로 외부 노출 채널에는 별도 toxic·bias 필터를 의무화한다. 4) 경량화 실측 : Q4_K_M GGUF에서의 한국어 응답 품질 저하 정도와 추론 속도(token/s)를 동일 인프라에서 측정. 5) 라이선스 트랙 : 기술 검증과 병행해 LG AI Research와 commercial license 협의를 조기에 시작해야 운영 전환 시점에 병목이 생기지 않는다. ④ 차순위 검토 후보 — 본 모델로 검증한 결과 한국어 품질이 더 필요하다면 EXAONE 3.5 7.8B(같은 라이선스), 라이선스 자유도가 더 필요하다면 Kakao Kanana 시리즈(Apache 2.0) 또는 Qwen 2.5 7B(Apache 2.0)를 비교군으로 검토할 것을 권한다.
