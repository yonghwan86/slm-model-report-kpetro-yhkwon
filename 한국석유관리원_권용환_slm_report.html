01모델 선택 이유

공공기관의 디지털 전환(AX) 과정에서 가장 중요한 요소는 데이터 보안과 자원 효율성입니다. Microsoft에서 개발한 Phi-3-mini 모델은 3.8B라는 소규모 파라미터임에도 불구하고, 고품질의 학습 데이터를 통해 7B~13B급 모델에 필적하는 추론 성능을 보여줍니다. 특히 외부 망 접속이 제한된 온프레미스(On-premise) 환경에서 가벼운 하드웨어 자원만으로 내부 매뉴얼 분석이나 RAG 기반의 지능형 업무 지원 시스템을 구축하기에 최적의 모델이라 판단하여 선정했습니다.

03학습 데이터와 튜닝 방식

Phi-3-mini는 약 3.3조 개의 토큰으로 사전학습되었으며, 단순 웹 크롤링 데이터가 아닌 '교과서급 고품질 데이터(Textbook-quality)'와 생성형 AI를 활용한 '합성 데이터(Synthetic data)'를 조합하여 모델의 논리적 사고력을 극대화했습니다. 튜닝 방식으로는 지도 미세 조정(SFT)과 직접 선호 최적화(DPO) 기법을 적용하여, 사용자의 복잡한 지침을 정확히 수행하고 안전한 답변을 생성하도록 최적화되었습니다.

05모델 파일 구성

ㅇ 저장소의 주요 파일들은 다음과 같은 핵심 기능을 담당합니다.
- config.json: 모델 아키텍처(레이어 수, 헤드 수 등) 및 활성화 함수 설정을 포함합니다.
- model.safetensors: 실제 모델의 학습된 가중치 파일로, 기존 방식보다 보안성과 로딩 속도가 뛰어납니다.
- tokenizer.json: 텍스트를 모델이 처리할 수 있는 토큰 단위로 변환하는 규칙을 정의합니다.
- generation_config.json: 답변 생성 시의 온도(Temperature)나 샘플링 전략 등 하이퍼파라미터 기본값을 저장합니다.

06활용 가능 업무

- 행정 문서 자동 요약: 대내외 보고서나 긴 회의록의 핵심 요점만을 빠르게 추출하여 업무 효율을 높입니다.
- RAG 기반 내부 지식 서비스: 석유 관련 법령집이나 시험기기 수리 매뉴얼을 데이터베이스화하여 직원들의 질문에 즉각 답변하는 지능형 헬프데스크 구축이 가능합니다.
- 업무 자동화 스크립트 지원: 엑셀 수식 생성이나 데이터 분석용 SQL 쿼리 초안 작성을 보조하여 IT 비전공 직원의 생산성을 향상시킵니다.

07한계와 주의사항

- 한국어 성능 제약: 영문 데이터 위주의 학습으로 인해 복잡한 한국어 문맥 추론 시 성능이 저하될 수 있으며, 별도의 한국어 파인튜닝이 필요할 수 있습니다.
- 컨텍스트 길이: 4K 길이는 방대한 양의 전문 서적 전체를 한 번에 처리하기에 부족할 수 있어, 효율적인 텍스트 분할(Chunking) 전략이 필수적입니다.
- 할루시네이션(환각): 학습 데이터 컷오프 이후의 최신 정보에 대해서는 부정확한 정보를 제공할 수 있으므로 최종 확인 절차가 필요합니다.

08종합 의견

Phi-3-mini는 작지만 강력한 성능을 가진 '가성비' 높은 모델로, 대규모 GPU 인프라 구축이 어려운 공공기관 단위 업무에 도입하기에 최적입니다. 특히 석유 유통 관리나 검사 정보 분석과 같은 전문 영역에서는 해당 도메인 데이터를 활용한 미세 조정(Fine-tuning)을 거친다면, 실제 현장 직무를 보조하는 강력한 '버디(Buddy)' 역할을 충분히 수행할 것으로 기대됩니다.
